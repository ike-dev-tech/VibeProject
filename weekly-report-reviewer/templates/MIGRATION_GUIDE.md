# 週報テンプレート v2 移行ガイド

## 変更の概要

旧テンプレートから新テンプレート（v2）への移行ガイドです。

### 主な変更点

| 項目 | 旧テンプレート | 新テンプレート（v2） |
|------|--------------|-------------------|
| セクション数 | 8個 | 6個 |
| デイリー記録 | なし | テーブル形式 |
| 振り返り | Good/Bad形式 | 4つの質問形式 |
| フレームワーク | なし | KPT |
| 継続性 | なし | 前週自動引き継ぎ |
| AI出力先 | 「AIからの総括（振り返り）」 | 「AIサマリ」 |

---

## 移行方法

### 方法1: 新しい週から切り替え（推奨）
次の週（例: 2026-W03）から新テンプレートを使用開始

**手順:**
1. `templates/weekly-template-v2.md` をコピー
2. ファイル名を `2026-W03.md` に変更
3. Obsidian Vaultの週報フォルダに配置
4. 以降、このテンプレートを使用

### 方法2: 既存週報を変換
現在の週報を新フォーマットに変換

**手順:**
1. 既存の内容を保存
2. 新テンプレートに手動で転記
   - 「今週やったこと」 → デイリーログにまとめる
   - 「Good/Bad」 → 4つの質問に変換
   - 「来週の目標」 → KPTのTryに統合

## セクション対応表

| 旧セクション | 新セクション | 備考 |
|------------|------------|------|
| ■今週自分が得たい結果 | ## 今週のフォーカス | 1つに絞る |
| ■今週のToDo | （削除） | デイリーログで管理 |
| ■今週やったこと ＆ 気づき | ## デイリーログ | テーブル形式に |

| ■来週の目標 | ## KPT（Try） | KPTに統合 |

| ▼年度目標 | ## 年度目標 | そのまま |

## 新テンプレートの使い方

### 1. デイリーログ（毎日1行）
毎日寝る前に1分で記録

```markdown
| 月 | 案件評価アプリのAPI実装完了 | 4/5 |
```

### 2. 振り返り（週末に5分）
4つの質問に各1文で答える

```markdown
1. **一番の成果は？** → 案件評価アプリのMVP完成
2. **なぜうまくいった？** → 朝の集中時間を確保できたから
3. **一番の障害は？** → デプロイ時のエラー対応に時間がかかった
4. **どう乗り越える？** → エラーログを事前に調査する習慣をつける
```

### 3. KPT（週末に3分）
3要素をシンプルに

```markdown
- **Keep（続ける）**: 朝の集中時間の確保
- **Problem（課題）**: デプロイプロセスの不理解
- **Try（来週試す）**: デプロイ手順のドキュメント化
```

### 4. 前週からの引き継ぎ（自動）
AIが前週のTry/Problemを自動挿入するので何もしなくてOK

## よくある質問

### Q: デイリーログを毎日書き忘れたら？
A: 週末にまとめて思い出して書いてもOKです。完璧を目指さず、続けることを優先してください。

### Q: 旧テンプレートの週報はどうする？
A: そのまま残してOKです。AIレビューシステムは新旧両方に対応します（後方互換性あり）。

### Q: 「調子」の数値はどう決める？
A: 主観でOKです。1=最悪、3=普通、5=最高のイメージで記録してください。

### Q: 年度目標は毎週コピペ？
A: はい。毎週意識するために残しています。変更があれば随時更新してください。

## トラブルシューティング

### AIレビューが動かない
- ファイル名が `2026-W02.md` 形式になっているか確認
- セクション名が正確か確認（`##` の数、スペースに注意）

### 前週の引き継ぎが表示されない
- 前週の週報ファイルが存在するか確認
- 前週のKPTセクションが正しく記入されているか確認
